Capabilities

One agent. Full-machine reach.

SHIMS isn't a chatbot bolted onto a search box. It's an operating shell with real tools, real memory, and real guardrails.

🗂️

Files & filesystem

Read, write, edit, search and organise files anywhere you allow — with path-safety guards and approval gates on anything destructive.

⌨️

Shell & code

Runs shell commands and executes Python/JavaScript in a sandbox. Safe, read-only commands run instantly; risky ones ask first.

🌊

Parallel wave engine

Independent tool calls fire in parallel waves with retries, budgets and deduplication — turning multi-step tasks from O(n) into O(waves).

🧠

Skills & memory

Procedural-memory "skills" teach SHIMS your conventions once and inject them automatically. Long sessions stay coherent via smart context management.

🌐

Web access

Search the internet and fetch pages on demand, then reason over the results — without leaving the conversation.

🎙️

Voice & desktop coworker

Talk to SHIMS hands-free, or summon it anywhere with a global hotkey via the desktop bridge — system tray, clipboard nudges, notifications.

Run fully local with Ollama, or plug in OpenAI, Anthropic, Gemini and HuggingFace. Switch models per task; keep sensitive work on-device.

🏭

Enterprise modules

♻️

Self-improvement

SHIMS learns from each run, refines its own plans and skills, and (with approval) can extend its own capabilities over time.

How it works

From request to result in four moves.

Every turn runs the same disciplined loop — so you always see what it's doing and stay in control.

Plan

SHIMS reads your goal and decides which tools to call right now — as a structured wave, not a guess.

Execute

Independent tools run in parallel with retries and budgets. Risky actions pause for your one-tap approval.

Reflect

Results flow back into context. SHIMS checks progress, recovers from failures, and plans the next wave.

Deliver

When the goal is met it stops and reports clearly — with a full, auditable trail of every action taken.

Security & trust

Powerful by design. Safe by default.

Autonomy you can actually deploy. SHIMS pairs broad capability with hard guardrails and a clear paper trail.

🔒

Local-first & private

Runs on your machine and defaults to local models. Nothing is sent to a third party unless you explicitly choose a cloud provider.

🛡️

Gated execution

Destructive file and shell actions are risk-classified and require explicit approval. Path-traversal and forbidden directories are blocked.

🧾

Audit & rate limits

Security-relevant events are logged, secrets are strength-checked, and built-in rate limiting keeps automation from running away.
